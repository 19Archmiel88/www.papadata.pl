# Incident Response

## Definicja incydentu
- niedostępność aplikacji
- błąd uniemożliwiający podstawowe flow
- masowe time-outy AI
- potencjalny wyciek danych lub PII

## Role
- Incident Commander
- Engineering
- Comms/Support

## Proces
1) Triage (5–15 min)
2) Mitigacja (rollback/feature flag)
3) Komunikacja (status page / mail)
4) Postmortem (co, dlaczego, jak zapobiec)

## Postmortem
- timeline
- root cause
- action items
